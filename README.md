# 李子生活陪伴插件

`astrbot_plugin_lizi_life` 是一个面向 AstrBot 的轻量生活陪伴插件，提供每日角色状态、基于知识库的多人群聊与每日事件、饮食建议、启动任务、成就、日记、晚安模式和服务器健康检查。

插件直接调用 AstrBot 当前会话所使用的聊天模型，例如 DeepSeek，不需要再次填写模型 API Key。

从 `0.3.0` 开始，插件默认不再向普通聊天强行注入自己的角色提示词。主聊天人设应在 AstrBot 自带的人格功能中维护；`/群聊` 和 `/今日事件` 的多角色资料则来自用户选择的 AstrBot 知识库。

## 环境要求

- AstrBot `>= 4.9.2` 且 `< 5`
- 已在 AstrBot 中配置可用的聊天模型
- Python 依赖由 AstrBot 根据 `requirements.txt` 自动安装
- 支持个人微信、企业微信、微信公众号、OneBot 和 Satori 等适配器

## 安装

进入 AstrBot WebUI：

1. 打开“插件管理”。
2. 选择通过 GitHub 仓库安装插件。
3. 输入以下地址：

```text
https://github.com/dadaliu484-pixel/astrbot_plugin_lizi_life
```

4. 等待依赖安装完成。
5. 打开插件配置，选择用于角色设定的 AstrBot 知识库，并按需修改生活偏好。
6. 重载插件；如果命令没有出现，重启一次 AstrBot。

## 命令总览

| 命令 | 功能 |
| --- | --- |
| `/李子帮助` | 查看插件命令列表 |
| `/状态` | 查看角色今日状态 |
| `/群聊 内容` | 模拟多名角色围绕指定内容聊天 |
| `/吃什么 [预算]` | 根据预算和饮食偏好生成饮食建议 |
| `/抽任务` | 抽取一个低压力启动任务 |
| `/完成` | 完成当前任务并累计进度 |
| `/成就` | 查看已经解锁的成就 |
| `/记录 内容` | 主动记录一件今天发生的事 |
| `/日记` | 生成或查看今天的日记摘要 |
| `/昨日回忆` | 生成或查看昨天的日记摘要 |
| `/今日事件` | 生成当天的角色关系小事件 |
| `/睡觉` | 开启晚安短回复模式 |
| `/起床` | 关闭晚安短回复模式 |
| `/status` | 查看云服务器和外部服务状态 |

## 功能详解

### 1. 今日状态

发送：

```text
/状态
```

插件会为当前用户生成当天固定的角色状态，例如：

```text
李子今日状态

心情：有点困
占有欲：72
嘴硬程度：89
想你程度：96
今日关键词：别熬夜
```

同一用户在同一天重复查询会得到相同状态，第二天自动更新。不同用户的状态互不影响。

启用 `inject_daily_state` 后，今日状态还会作为临时上下文提供给普通聊天模型，让角色回复和当天状态保持一定一致性。该上下文不会写入聊天历史。

### 2. 多角色群聊模拟

发送：

```text
/群聊 我今天又刷了两个小时视频
```

插件会先用本次群聊主题检索 `role_knowledge_bases` 中选择的 AstrBot 知识库，再把检索到的人物设定和关系资料交给当前聊天模型生成群聊。

只有知识库检索到的角色与关系才能用于生成。插件会要求模型不要补写知识库中不存在的共同经历。没有选择知识库或没有检索到相关资料时，命令会直接提示补充知识库，不再使用内置的简化角色设定冒充正式人设。

这只是一次模型生成，不会真的创建多个机器人或多个模型会话。

### 3. 饮食建议

发送：

```text
/吃什么
/吃什么 15
/穷鬼套餐 20
```

不填写预算时默认按 20 元生成建议。预算会限制在 5 至 500 元之间，避免异常输入。

插件会结合 `food_preferences` 中的忌口、偏好和常备食材，生成午饭、晚饭和简短理由。建议主要用于日常选择，不提供医疗诊断，也不会虚构精确营养数据。

### 4. 启动任务

发送：

```text
/抽任务
```

插件会从 `starter_tasks` 任务库中抽取一个低压力任务，并生成难度、预计耗时和奖励：

```text
今日任务卡：去洗脸并喝一杯水

难度：★
耗时：3 分钟
奖励：李子少骂你一次

任务内容：
去洗脸并喝一杯水
```

任务库可在 WebUI 中编辑，每行一个任务。该功能适合解决“知道要做事，但暂时启动不了”的情况。

### 5. 完成任务

完成任务后发送：

```text
/完成
```

插件会将当前任务标记为完成，增加累计完成次数，并记录到当天活动中。如果没有待完成任务，会提示先使用 `/抽任务`。

每次只维护一个当前任务。再次抽取会替换尚未完成的旧任务。

### 6. 成就系统

发送：

```text
/成就
```

当前可解锁的成就包括：

- 完成第一个小任务
- 累计完成 3 个任务
- 累计完成 10 个任务
- 生成第一篇日记
- 完成第一次服务器巡检

成就根据真实保存的插件数据自动计算，不需要手动领取。

### 7. 主动记录

发送：

```text
/记录 今天把房间收拾了一小块
```

插件会把内容写入当天活动列表，供 `/日记` 使用。建议只记录希望进入日记摘要的事情。

插件自身的一些操作也会写入活动列表，例如抽任务、完成任务、查看服务器状态和切换晚安模式。

### 8. 今日日记

发送：

```text
/日记
```

插件会根据当天的主动记录和插件活动生成一篇简短日记。日记只总结已有事件，不应凭空补充用户没有说过的经历。

同一天首次生成后会保存摘要；再次查询直接返回已保存版本，不会反复调用模型或改变内容。

如果当天没有足够记录，插件会提示先使用 `/记录`。

### 9. 昨日回忆

发送：

```text
/昨日回忆
```

功能与 `/日记` 相同，但目标日期是昨天。已经生成过的昨日摘要会直接读取；没有记录时会给出提示。

### 10. 今日事件

发送：

```text
/今日事件
```

插件会从吃饭、熬夜、服务器、出门和整理房间等生活主题中选择一个，检索角色知识库，然后生成 50 至 100 字的日常小事件。

同一用户每天只生成一个事件，生成后保存，当天重复查询不会变化。

事件只能使用知识库中存在的角色、关系与世界观。知识库资料不足时会提示补充，不会静默编造。

### 11. 晚安模式

发送：

```text
/睡觉
```

插件会开启当前用户的晚安模式，并回复一条简短晚安消息。开启后，普通聊天请求会临时附加 `night_prompt` 中的规则，使回复更短，避免主动展开长话题。

晚安模式不会禁止正常回答紧急或重要问题。

### 12. 起床模式

发送：

```text
/起床
```

插件会关闭晚安模式，普通聊天恢复正常风格。`/晚安` 和 `/早安` 分别是 `/睡觉` 和 `/起床` 的别名。

### 13. 云服务器状态

发送：

```text
/status
```

插件会返回运行 AstrBot 的云服务器状态：

- CPU 使用率
- 内存使用率和可用内存
- 磁盘使用率和可用空间
- 系统持续运行时间
- 配置的外部健康检查结果

这里检测的是插件所在的云服务器，不是用户本地电脑。

### 14. FRP 外部健康检查

通过 `health_endpoints` 可以让云端 AstrBot 检查 FRP 隧道后方的本地服务，例如 embedding 服务或 LM Studio。

每行格式：

```text
名称|健康检查URL|可选令牌
```

示例：

```text
本地 Embedding|https://example.com/embedding/health|my-secret-token
LM Studio|https://example.com/lmstudio/v1/models|my-secret-token
```

如果填写第三段，插件会发送：

```text
Authorization: Bearer my-secret-token
```

请求超时时间由 `health_timeout` 控制，默认 5 秒。HTTP 2xx 和 3xx 被视为正常，其他状态码或连接错误会显示为异常。

不要把 LM Studio、embedding API 或其他本地服务无鉴权暴露到公网。推荐只暴露一个信息有限的健康检查接口，并同时使用令牌、反向代理访问控制和 FRP 端口限制。

## WebUI 配置

| 配置项 | 说明 |
| --- | --- |
| `character_name` | 主角色名称，默认为“李子” |
| `character_prompt` | 插件内部生成日记时使用的补充角色设定 |
| `role_knowledge_bases` | `/群聊` 和 `/今日事件` 使用的 AstrBot 角色知识库 |
| `knowledge_top_m` | 每次知识库检索最终取回的片段数量 |
| `starter_tasks` | `/抽任务` 使用的任务库 |
| `food_preferences` | 饮食偏好、忌口和常备食材 |
| `health_endpoints` | FRP 后方服务的健康检查地址 |
| `health_timeout` | 每个健康检查的超时时间 |
| `night_prompt` | 晚安模式对普通聊天的回复约束 |
| `inject_daily_state` | 是否向普通聊天临时注入今日状态，默认关闭 |
| `inject_character_prompt` | 是否让插件覆盖普通聊天角色提示，默认关闭且不建议开启 |

修改配置后建议在插件管理页面重载插件。

## 人设与知识库的正确分工

### AstrBot 人格

AstrBot 自带人格负责主聊天中的“她是谁、如何说话、如何称呼用户、有哪些行为边界”。建议只维护一份主人格，不要同时在多个插件里重复写完整人设。

主人格适合包含：

- 角色身份和核心性格
- 语言风格与回复长度
- 对用户的称呼
- 明确的行为边界
- 不确定信息的处理方式

不要在主人格中堆大量每日事件、聊天历史或会频繁变化的数值。

### AstrBot 知识库

知识库负责可检索的稳定事实，例如多名角色的人物卡、关系网、共同世界观和已经确认的经历。`/群聊` 与 `/今日事件` 会按当前主题只取回相关片段。

推荐将资料拆成多个短文档或结构清晰的段落：

```text
标题：角色卡 - 李子
身份：……
核心性格：……
说话风格：……
习惯称呼：……
喜欢与讨厌：……
行为边界：不羞辱用户，不以嫉妒制造冲突，不声称拥有现实身体。
```

```text
标题：角色卡 - 小夏
身份：……
核心性格：……
说话风格：……
与李子的关系：……
与用户的关系：……
行为边界：……
```

```text
标题：关系网
李子与小夏：……
李子与阿岚：……
小夏与阿岚：……
三人已确认的共同经历：……
禁止默认存在的经历：没有记录的旅行、争吵、礼物或现实见面不得编造。
```

```text
标题：世界规则
故事发生的环境：……
角色是否知道自己是 AI：……
角色能否访问现实设备：……
允许出现的日常场景：……
禁止出现的设定：……
```

不建议把所有人物资料写成一个几万字的连续故事。较短、标题明确、事实重复较少的资料更容易被 embedding 准确检索。

## 人设异常排查

### 回复语气突然变化

先确认插件配置中的 `inject_character_prompt` 和 `inject_daily_state` 均为关闭。然后检查 AstrBot 当前会话实际选择的人格，而不只是全局默认人格。

### 多角色互相串台

检查知识库中的每个事实是否明确写出角色名称。避免只写“她比较活泼”，应写“小夏比较活泼”。人物卡之间不要出现互相矛盾的年龄、关系和称呼。

### 模型编造共同经历

在关系网文档中单独列出“已确认共同经历”和“禁止默认存在的经历”。降低知识库切片大小，并确保关系规则能作为完整片段被检索。

### 人设过度黏人或有控制欲

不要只写“占有欲强”“非常爱用户”。应同时写清可观察的表达方式和边界，例如“会用一句轻微吃醋的玩笑表达，但不阻止用户社交，不贬低现实朋友，不要求用户证明感情”。

### 同一问题每次设定不同

把稳定事实写入人格或知识库，不要依赖历史聊天让模型自己归纳。删除相互矛盾的旧知识片段后重新构建知识库索引。

## 数据保存与隐私

插件使用 AstrBot 提供的插件级 KV 存储保存：

- 当前任务与任务完成次数
- 已生成的日记摘要
- 最近 14 天的插件活动和主动记录
- 每日事件
- 晚安模式状态
- 成就所需的统计标记

插件不会额外保存完整聊天记录。`/群聊` 和 `/吃什么` 的输入会发送给当前配置的聊天模型进行生成，但不会作为独立插件数据长期保存。

动态状态和晚安模式规则使用临时上下文注入，不写入 AstrBot 会话历史。

## 常见问题

### 命令没有出现

在插件管理页面重载插件。如果依然无效，检查 AstrBot 日志中的依赖安装或版本错误，并重启 AstrBot。

### AI 功能只返回固定文字

这通常表示当前会话没有可用聊天模型，或者模型调用失败。检查 AstrBot 中的 DeepSeek 等提供商是否已经启用。

### `/status` 看不到本地 embedding

`/status` 默认只能看到云服务器。需要先通过 FRP 和带鉴权的健康检查接口暴露本地服务，再填写 `health_endpoints`。

### 更新插件后旧数据是否丢失

不会。数据保存在 AstrBot 插件 KV 存储中，而不是 GitHub 插件目录内。正常更新或重装代码不会覆盖这些数据。
